🛠️ Need help setting up Supabase? Show SQL & Setup Instructions ▼

To connect your planner, follow these steps in your Supabase project:

  1. Go to SQL Editor in the left sidebar and click + New Query.
  2. Copy and paste the SQL script below, then click Run:
-- Drop existing table if migrating DROP TABLE IF EXISTS daily_activities CASCADE; -- Create secure multi-user daily activities table CREATE TABLE daily_activities ( user_id UUID NOT NULL REFERENCES auth.users(id) DEFAULT auth.uid(), date DATE NOT NULL, period_number INTEGER NOT NULL, class_section TEXT DEFAULT '', classwork TEXT DEFAULT '', homework TEXT DEFAULT '', photo_url TEXT DEFAULT '', created_at TIMESTAMPTZ DEFAULT timezone('utc'::text, now()), PRIMARY KEY (user_id, date, period_number) ); -- Enable Row Level Security (RLS) ALTER TABLE daily_activities ENABLE ROW LEVEL SECURITY; -- Allow authenticated/anon users to read their own data CREATE POLICY "Allow users to read own data" ON daily_activities FOR SELECT TO authenticated, anon USING (auth.uid() = user_id); -- Allow authenticated/anon users to modify their own data CREATE POLICY "Allow users to modify own data" ON daily_activities FOR ALL TO authenticated, anon USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
  1. Go to Storage in the left sidebar, click New Bucket. Name it activity_photos and make it Public.
  2. Under Storage, go to Policies, find activity_photos bucket, click New Policy, choose For full customization, name it, select operations ALL, target roles anon, authenticated, and set both USING and WITH CHECK expressions to true.
  3. To enable Google login, navigate to Authentication > Providers > Google, toggle it on, and paste your Google OAuth Client ID and Secret (obtainable from Google Cloud Console).
  4. Go to Project Settings > API, copy your Project URL and anon public API key, and paste them into the fields above.
